Kung-Fu: Agentic IDE Tool

Deploy rules, skills and agent persona files from trusted source folders into multiple agentic IDE's (workspaces).

Get Kung-Fu from Microsoft Store

The Problem:

Working in more than one agentic IDE means you are maintaining two or more sets of Rules, Skills and Agent Persona folders. Working on more than one dev machine squares this irritation.

Everyone should have already packed up rules, skills and agent persona's into a Git Repo to make a source of truth and version agentic stuff. Context is source code!

So I decided I needed some kind of simple copy-paste tool with a comfy UI....

The Solution:

Kung-Fu does a simple copy-paste from a source of truth folder (Git Repo please!) into each IDE's settings folder.

With it, every IDE feels like home, so I can check out the new hotness from Anthropic or Google or whoever.

What it deploys

  • Agentic IDE rules files.
  • Reusable agent skills.
  • Agent persona files.

Not a package manager.

IDE's like Claude Code, Codex etc... move fast! They already have Marketplaces and Plugins to handle agentic artefacts with richer lifecycles. If something is more complicated than the copy-paste of a handful of files to the right destination, Kung-Fu is not the tool for the job by design.

Agentic artefacts that have to be integrated into a settings.json file for your project, or machine are beyond the scope of Kung-Fu:

  • Guardrails are not handled.
  • Hooks are not touched.
  • Validators are not managed.
  • Network transfers between machines are not done.

Local-first

Kung-Fu runs locally on your PC.

It does not require an account or a publisher-operated cloud service or other riff raff that gets in the way.

Kung-Fu works with the source folder and the destination directories you choose.

Why Windows Store?

1. Easy to for you to get it installed.

2. Microft's app certification process for trust and validation.

3. Automatic Updates.

System requirements

Kung-Fu is available for x64 Windows PCs running Windows 10 version 17763.0 or later.
